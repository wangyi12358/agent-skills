# example-skill

## Purpose

提供一个最小可用示例，帮助你快速开始编写自己的 skill。

## When to use

- 当用户提出一个适合固化为标准流程的任务时
- 当你希望把常见任务沉淀成可复用说明时

## Inputs

- 用户目标
- 当前项目上下文
- 任何必要的限制条件

如果信息不完整，先读取仓库上下文，再基于合理默认值继续。

## Workflow

1. 识别任务目标。
2. 找到相关文件或上下文。
3. 按照既定流程执行。
4. 给出简洁结果和下一步建议。

## Output

- 清楚说明做了什么
- 标出涉及的文件路径
- 在必要时给出验证方式

## Constraints

- 不要做破坏性操作，除非用户明确要求
- 尽量遵循仓库已有约定

## Examples

### Example 1

**User request**

```text
帮我检查这个仓库里的 API 命名是否一致
```

**Expected behavior**

- 搜索相关接口定义
- 归纳命名模式
- 指出不一致之处并给出修改建议
